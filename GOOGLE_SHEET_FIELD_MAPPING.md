## 🎯 Quick Reference: What Goes Where

### Direct Field Mapping
```
Google Sheet          →    CRM Database Field
───────────────────────────────────────────────
full_name             →    leads.full_name
phone_number          →    leads.phone
phone_number          →    leads.whatsapp (duplicate)
email                 →    leads.email
country               →    leads.country
your_highest_...      →    leads.qualification
in_which_program...   →    leads.course_interested
platform              →    leads.source
```

### Auto-Generated by CRM
```
CRM Field                Value
─────────────────────────────────────
lead_id                  "LEAD00123" (auto-incrementing)
status                   "New Lead"
ai_score                 0-100 (calculated by AI)
ml_score                 0-1 (ML model score)
conversion_probability   0-1 (AI prediction)
ai_segment               "Hot/Warm/Cold"
assigned_to              "Super Admin" (default)
hospital_id              null (Super Admin has no hospital)
created_at               Current timestamp
updated_at               Current timestamp
```

### Metadata in Notes
```
All campaign fields saved in first note:
- Platform
- Campaign Name
- Ad Name
- Ad Set Name
- Form Name
- Created Time
- Is Organic
- Import timestamp
```

## 🔧 How Apps Script Maps Fields

### In Code (for reference):

```javascript
const lead = {
  // Direct mappings
  full_name: row['full_name'],
  phone: row['phone_number'],
  email: row['email'],
  country: row['country'],
  qualification: row['your_highest_qualification'],
  course_interested: row['in_which_program_are_you_interested_?'],
  source: row['platform'],
  
  // Duplicate phone to WhatsApp
  whatsapp: row['phone_number'],
  
  // Optional (null if not applicable)
  branch: null,
  company: null,
  assigned_to: null, // Will default to Super Admin
  
  // Campaign metadata goes in notes
  notes_text: buildNotesFromMetadata(row)
};
```
